# ERCOT Dashboard Kickoff

Date: 2026-05
Category: Weekly Reflection

## What I am building

I am building an ERCOT power-market analytics portfolio to demonstrate practical desk-support skills: gathering market data, cleaning time-series records, building dashboards, explaining price movement, and documenting repeatable workflows.

The first project is an ERCOT Real-Time Market Dashboard. The initial version uses mock ERCOT-style data so I can focus on structure, dashboard design, price-spike logic, and written market summaries before connecting public ERCOT or GridStatus data.

## Why this project exists

I am transitioning from high school economics and psychology teaching into power-market analytics while completing a master's degree in data science. This portfolio is meant to show how my teaching background, economics communication, Python skills, SQL foundation, and data science training can translate into an analyst support role.

## What Project 1 will focus on

Project 1 will monitor hub and load-zone prices, day-ahead versus real-time spreads, load forecast error, and price-spike intervals. The goal is to answer practical questions: what is happening now, which areas are moving, what changed from day-ahead expectations, and what should a desk pay attention to today?

## What I need to be careful about

This project should be honest. Until live public ERCOT or GridStatus data is connected, all dashboard examples should be labeled as mock ERCOT-style data. The value of the first phase is not pretending to be a live trading system. The value is building a clean analyst workflow that can later accept real public market data.
